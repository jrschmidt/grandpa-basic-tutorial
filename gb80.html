Program lines in BASIC
      10 REM GRANDPA BASIC 1980 - EXAMPLE 1
      20 PRINT "HELLO WORLD"

In the original BASIC programming language, each statement in the program was placed on a line preceeded by a line number.

      220 X=7

Program lines in BASIC are executed in numerical order, except when statements such as GOTO, IF, GOSUB or RETURN divert program flow elsewhere. To write a BASIC program, just type one line at a time on the BASIC console. Place the desired line number in front of the line as you type. After entering a line, hit ENTER. The console will tell you if there is a syntax error in the line.

Line numbers can be any integer from 1 to 32767. This number, which is 1 less than 32K (1024 times 32), was the maximum line number on many of the original BASIC platforms.

The maximum number of characters for a line on the Grandpa BASIC 1980 console is 80. This includes the line number. Lines do not continue over to the next line.

Be sure to follow the syntax exactly. Some habits that are common with modern programmers will not work here. For example, it is common today to place spaces around an equals sign, like this:

      a = 10

This will not work in BASIC. If you try to type

      A = 10

instead of

        A=10

you will get a syntax error.

To correct a line while you are typing, you can use the BACKSPACE key and retype. You cannot move the console cursor with arrow keys or mouse clicks. If you wish to change a line that has already been entered, just type in a new line with the same number. It will replace the original line that had that number. To see your changes, type the LIST command.

To insert one or more lines between two other lines, give the new line or lines a number that is between the other two line numbers.

      220 A=371
      230 C=687

If we type

      225 B=526

we will get this result.

      LIST
      220 A=371
      225 B=526
      230 C=687

For this reason, it is common when programming in BASIC to space out line numbers by 10 or some other increment. This will allow you to insert lines without renumbering everything.

To remove a line number entirely, type the number, then hit ENTER.

Typing CLEAR will remove all program lines and let you start over.

And remember that you can type LIST to see your program at any time.

Variables in BASIC
      10 REM GRANDPA BASIC 1980 - EXAMPLE 2
      20 $H="HELLO WORLD"
      30 P=3.1416
      40 Z4=1980
      50 $S2="DOLLARS"

There are two types of variables in Grandpa BASIC 1980, numeric and string. A variable name with a dollar sign ($) in front ($A) is a string variable. A variable name without the dollar sign (A) is a numeric variable.

In Grandpa BASIC 1980, as well as the earliest versions of BASIC, variable names consist of either a single, upper case letter of the alphabet (A-Z), or a single letter followed by a single digit (0-9). Thus, A, A2, X, X0 and W9 are all legal names for numeric variables, and $S, $Q3, and $M are legal names for string variables

All variables in BASIC have global scope. There are no local variables. BASIC predates object oriented languages and languages with methods or functions.

Arrays, also known as matrices in BASIC, were part of the BASIC language from the beginning. They have not been implemented yet in Grandpa BASIC 1980.

Assignment Statements
      10 REM GRANDPA BASIC 1980 - EXAMPLE 3
      20 X=1
      30 P=3.1416
      40 N=N+1
      50 T=T0+T1+T2+T3
      60 R=((Z8*Z7)-Z4)/(D9*H3^2)
      70 $S2="DOLLARS"
      80 $M="MY NAME IS "+$N1

Assignment statements in BASIC are written with a variable name, followed by an equals sign (=), followed by an expression defining the value to be assigned to the variable. Do not use spaces around the equals sign, as this is a modern programming convention that was not used back then.

      20 X=1
      70 $S2="DOLLARS"

The part on the right side of the equals sign is the expression. This is the value to be assigned to the variable. An expression can be simple or compound.

The very earliest versions of BASIC (Dartmouth BASIC and others) called these assignment statements "LET statements". The syntax required the keyword LET at the beginning of the assignment (eg., LET X=Y*Z). This requirement was soon deprecated.

Numeric assignment and expressions

A simple numeric expression consists of just a number. The number can be a simple integer, or contain a decimal.

      610 I=0
      620 N=1
      630 M9=32767
      640 P=3.1416
      650 G1=0.0025
      660 V9=42.0388

You can also assign the value of one variable to another.

      670 X=Y

Compound numeric expressions can be formed by combining numbers and variables with arithmetic operators and grouping them with parentheses

      710 Z=A+B+C+D
      720 X=R2/R7
      730 T1=M^3-(40*U)/(J4+1)

Available arithmetic operators are Addition, Subtraction, Multiplication, Division and Exponentiation (+, -, *, / and ^) 

      810 M=J+K
      820 I=I-1
      830 F=F1*F2
      840 G=D/500
      850 W=C^3
String assignment and expressions

A simple string expression consists of zero or more characters surrounded by double quote marks ("). Using single quotation marks (') will cause a syntax error.

      910 $H=""
      920 $E3="   "
      930 $R="Y"
      940 $M3="WATER"
      950 $T="BIRDS ARE SINGING IN THE TREES."

Just like numeric assignment, you can assign the value of one string variable to another.

      960 $B=$A

Compound string expressions can be formed by concatenating two or more string values into one, using the plus sign (+). By string values we mean either variable names or strings in quotes.

      970 $N=$N1+$N2+$N3
      980 $V="MY NAME IS "+$N
REM
Comment Lines
      10 REM GRANDPA BASIC 1980 - EXAMPLE 4
      20 REM
      30 REM GOOD PROGRAMMERS USE COMMENTS
      40 REM

Comments were called remarks in BASIC. Any line with a line number followed by REM is a comment line. Any text following the REM, out to the end of the line, is the comment.

      300 REM
      310 REM THIS IS THE COMPUTATION SECTION
      320 REM HERE WE COMPUTE THE SUM
      330 REM OF X AND Y
      340 REM

A REM statement without any text can be used as a spacer line.

      400 REM
GOTO
Unconditional transfer of program flow
      10 REM GRANDPA BASIC 1980 - EXAMPLE 5
      20 PRINT "BEGINNING"
      30 GOTO 60
      40 PRINT "END"
      50 GOTO 100
      60 PRINT "MIDDLE"
      70 GOTO 40
      100 PRINT "DONE"

GOTO statements are very simple. When BASIC executes a GOTO statement, it transfers program flow to the designated line number.

      8800 GOTO 2000
IF
Conditional transfer of program flow
      10 REM GRANDPA BASIC 1980 - EXAMPLE 6
      300 IF A=B THEN 600

IF statements transfer program flow based on the value of an expression that evaluates to TRUE or FALSE. These expressions are called boolean expressions.

Boolean expressions in Grandpa BASIC 1980 IF statements compare a variable to another variable or value. Allowable comparators for numeric expressions are

  • equals =
  • not equal <>
  • greater than >
  • greater or equal >=
  • lesser than <
  • lesser or equal <=
      410 IF H=22 THEN 1800
      420 IF G<>H THEN 2300
      430 IF N>100 THEN 2600
      440 IF I>=8 THEN 2900
      450 IF T<7 THEN 3200
      460 IF S<=10 THEN 3500

The only allowable boolean operators in Grandpa BASIC 1980 for string values are

  • equals =
  • not equal <>
        510 IF $Q="YES" THEN 4400
        520 IF $G<>$H then 4700

The left side of the equals sign must be a variable name (numeric or string). The right side can be another variable name, or a numeric or string literal value (62480, 327.42, "Y", "MISSOURI", ). Left and right sides must be both numeric values or both string values.

        610 IF Q1=Q2 THEN 5100
        620 IF V=0 THEN 5300
        630 IF $T=$T8 THEN 5500
        640 IF $K="SATURDAY" THEN 5700
GOSUB & RETURN
Transfer program flow and return
      10 REM GRANDPA BASIC 1980 - EXAMPLE 7
      200 PRINT "BEFORE THE SUBROUTINE"
      220 GOSUB 6000
      240 PRINT "AFTER THE SUBROUTINE"
      990 END
      6000 REM
      6010 REM EXAMPLE SUBROUTINE
      6020 A=1776
      6030 B=3321
      6040 Z=A+B
      6090 RETURN

GOSUB and RETURN statements are used to transfer program flow to another part of the code, execute that code, then transfer program flow back to the original section of code. After the RETURN statement, program execution resumes at the line after the line that called the subroutine.

The following code:

      10 PRINT "AAA"
      20 PRINT "BBB"
      30 PRINT "CCC"
      100 GOSUB 500
      110 PRINT "GGG"
      120 PRINT "HHH"
      130 PRINT "III"
      200 END
      500 PRINT "DDD"
      510 PRINT "EEE"
      520 PRINT "FFF"
      530 RETURN

Will give the following output:

      AAA
      BBB
      CCC
      DDD
      EEE
      FFF
      GGG
      HHH
      III
INPUT
Accept user input from the keyboard
      10 REM GRANDPA BASIC 1980 - EXAMPLE 8
      40 INPUT X
      50 INPUT "WHAT IS YOUR NAME";$N

INPUT statements allow you to accept user input from the keyboard, which is then assigned to program variables. INPUT statments can be used for both numeric variables and string variables.

All that's necessary is the INPUT keyword followed by the variable name.

      600 INPUT W5
      610 INPUT $T

Optionally, you can specify text for the input prompt. Place the text for the prompt in quotes, followed by a semicolon (;), then a variable name.

      620 INPUT "HOW MANY";N
      630 INPUT "CITY";$C2
PRINT
Print output on the console
      10 REM GRANDPA BASIC 1980 - EXAMPLE 9
      50 PRINT " "
      60 PRINT "THE CURRENT VALUE FOR PRODUCT ID"
      70 PRINT $P6
      80 PRINT "IS"
      90 PRINT V

PRINT statements write program output to the console. The PRINT keyword is followed by a string value, either a string variable name, or a string literal in quotes

The PRINT keyword followed by two quotation marks separated with a single blank space, PRINT " ", can be used to print a blank line.

END
Terminate program execution
      10 REM GRANDPA BASIC 1980 - EXAMPLE 10
      900 PRINT "FINISHED"
      990 END

Program execution stops when an END statement is reached.

END statements are optional and are not required. However, they can be useful for implementing multiple termination points. You can have more than one END statement, reached by different paths.

      640 IF B>2000 THEN 800
      700 REM END WITH A LITTLE ONE
      710 PRINT "THAT WAS A LITTLE ONE"
      790 END
      800 REM END WITH A BIG ONE
      810 PRINT "THAT WAS A BIG ONE"
      890 END

LISTING 8
Guess a number

This program selects a random number, then tells you if each guess is 'too low' or 'too high'. It uses the RND and INT functions. RND gives a random number between 0.0 and 1.0, and INT strips the digits past the decimal point and returns an integer ("whole number").

      10 REM GRANDPA BASIC 1980 - LISTING 8
      20 REM GUESS A NUMBER
      100 M=1000
      200 X=INT(M*RND)
      300 PRINT " "
      310 INPUT "ENTER YOUR GUESS";G
      400 IF G=X THEN 900
      410 IF G>X THEN 800
      700 PRINT "TOO LOW"
      710 GOTO 300
      800 PRINT "TOO HIGH"
      810 GOTO 300
      900 PRINT "YOU WIN!"
      999 END
LISTING 9
Approximate Pi

This program approximates pi by using what is called a "Monte Carlo method". It emulates points picked at random inside a square, then forms an approximation of pi by computing the ratio of points that fall within a circle inscribed within that square.

      10 REM GRANDPA BASIC 1980 - LISTING 9
      20 REM APPROXIMATE PI WITH MONTE CARLO METHOD
      100 C=0
      110 I=0
      120 N=2000
      200 I=I+1
      210 IF I>=N THEN 800
      300 X=RND
      310 Y=RND
      400 R2=X*X+Y*Y
      410 IF R2>=1.0 THEN 200
      500 C=C+1
      510 GOTO 200
      800 P=(C*4)/N
      900 PRINT "OUR APPROXIMATION OF PI IS"
      910 PRINT P
      999 END

If you really want to get into the details, the algorithm as implemented here actually just computes points within a one quarter section of the circle and square.

You can change the value of N in Line 120. The more iterations you do, the more accurate your approximation of pi will be.

LISTING 10
Approximate square roots

This program will approximate square roots by successively adjusting "min" and "max" values until they are both within a certain distance of each other. In this case, if you use the values in this example, the square root will be estimated to 4 decimal places. Since 32 x 32 is greater than 1000, the square root of any positive number less than 1000 will be between 0 and 32.

      10 REM GRANDPA BASIC 1980 - LISTING 10
      20 REM APPROXIMATE SQUARE ROOTS
      100 A=0
      110 B=32
      200 PRINT "INPUT A NUMBER BETWEEN 0.0 AND 1000.0"
      210 INPUT X
      400 D=B-A
      410 IF D<=0.0001 THEN 800
      500 Z=(A+B)/2
      510 Z2=Z*Z
      520 IF Z2>X THEN 700
      600 A=Z
      610 GOTO 400
      700 B=Z
      710 GOTO 400
      800 PRINT "YOUR SQUARE ROOT IS APPROX"
      810 PRINT Z
      999 END
